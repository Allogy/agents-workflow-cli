# Design: `workflow run` Command (RAG-955)

## Summary

Add a `workflow run` command that starts Temporal workflow execution from the CLI,
displays node-by-node progress, and writes a `.last_run` context file for
subsequent HITL interaction commands (RAG-956).

## Command Interface

```
workflow run <identifier> [--input JSON|@file] [--stream] [--no-follow]
```

- **`identifier`**: UUID string or workflow name.
  Resolution order: UUID passthrough, lockfile lookup (`*.workflow.lock` in cwd), API name search.
- **`--input`**: JSON string or `@path/to/file.json` for initial inputs.
- **`--stream`**: Use SSE streaming instead of polling (default: polling).
- **`--no-follow`**: Fire-and-forget — start, print `run_id`, exit immediately.

## Execution Modes

### Polling (default)

1. Resolve identifier to workflow UUID.
2. Start via `POST /v2/workflows/{id}/start/temporal`.
3. Write `.workflow.last_run` immediately.
4. Poll `GET /v2/workflows/{id}/status?run_id={run_id}` every 2 seconds.
5. Display node-by-node progress with Rich.
6. On HITL gate: print hint, exit 0.
7. On COMPLETED: print summary, exit 0.
8. On FAILED: print error, exit 1.

### SSE Streaming (`--stream`)

1. Resolve identifier to workflow UUID.
2. Start via `POST /v2/workflows/{id}/run/temporal` with `Accept: text/event-stream`.
3. Write `.workflow.last_run` immediately.
4. Parse SSE `data:` lines from httpx streaming response.
5. Display events in real-time with Rich.
6. On HITL gate: print hint, exit 0.
7. On RUN_FINISHED: print summary, exit 0.
8. On RUN_ERROR: print error, exit 1.

### Fire-and-forget (`--no-follow`)

1. Resolve identifier, start workflow, write `.last_run`.
2. Print `run_id` and exit 0.

## `.last_run` Context File

Written to `.workflow.last_run` in the current working directory:

```yaml
# Auto-generated by workflow CLI. Do not edit manually.
workflow_id: 939843a8-6257-4475-bfc0-f7d6500d9f00
run_id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
instance: https://stage.sb.allogy.com
started_at: 2026-02-25T10:30:00Z
```

## Progress Display

### Polling mode

```
Running workflow: Invoice Processing (939843a8-...)
Run ID: a1b2c3d4-...

[1/4] plain_txt_input (upload_invoice) ... waiting for input
⏸  Workflow paused — waiting for input at node 'upload_invoice'
   Use: workflow input a1b2c3d4 --data '{"text": "..."}'
```

On completion:

```
[1/4] plain_txt_input (upload_invoice) ✓
[2/4] llm_call (extract_data) ✓
[3/4] human_review (review_results) ✓
[4/4] structured_output (final_output) ✓

✓ Workflow completed in 12.3s
```

### SSE mode

Events printed as they arrive:

```
▶ RUN_STARTED
⏳ STEP_STARTED  upload_invoice (plain_txt_input)
✓ STEP_FINISHED  upload_invoice
⏳ STEP_STARTED  extract_data (llm_call)
✓ STEP_FINISHED  extract_data
⏸ WAITING_FOR_REVIEW at review_results
   Use: workflow review a1b2c3d4 --approve
```

## Identifier Resolution

Order of resolution for the `<identifier>` argument:

1. **UUID passthrough**: If it matches UUID format, use directly.
2. **Lockfile lookup**: Scan `*.workflow.lock` files in cwd for matching name.
3. **API name search**: Call `list_workflows()` and filter by name (case-insensitive).

## File Structure

```
src/cli/commands/run.py      # Command implementation
src/cli/last_run.py          # .last_run read/write (shared with RAG-956)
tests/test_run_command.py    # Unit tests
```

`last_run.py` is a separate module so RAG-956 can import the reader.

## Scope Exclusions (RAG-956)

- `workflow status` command
- `workflow input` command
- `workflow review` command
- Reading `.last_run` for implicit run-id (only writing in this ticket)

## Dependencies

- RAG-946: API client with V2 Temporal endpoints (merged)
- RAG-944: CLI scaffolding (merged)
- httpx streaming support (already a dependency)

## API Endpoints Used

| Endpoint | Mode | Purpose |
|----------|------|---------|
| `POST /v2/workflows/{id}/start/temporal` | Polling, no-follow | Start execution |
| `GET /v2/workflows/{id}/status?run_id={id}` | Polling | Poll status |
| `POST /v2/workflows/{id}/run/temporal` | SSE | Stream execution |
| `GET /v1/workflows/` | Name resolution | Find workflow by name |
